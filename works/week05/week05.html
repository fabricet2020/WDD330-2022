Chapter 10: Testing and Debugging

Summary

Errors are caused when something goes wrong in a program. They are usually caused by one of the following:

  • System error ― there’s a problem with the system or external devices with which the program is interacting.
  • Programmer error ― the program contains incorrect syntax or faulty logic; it could even be as simple as a typo.
  • User error ― the user has entered data incorrectly, which the program is unable to handle.

Exceptions

  • An exception is an error that produces a return value that can then be used by the program to deal with the error.

Stack Traces

  • This is a sequence of functions or method calls that lead to the point where the error occurred.

Warnings

  • A warning can occur if there’s an error in the code that isn't enough to cause the program to crash.
  • Note that not all browsers will display a warning for the code
  • When a runtime error occurs in the browser, the HTML will still appear, but the JavaScript code will stop working in the background
  • Strict mode is the norm for programming. To use, include this in the first line: 'use strict';
  • Linting Tools such as JS Lint, JS Hint, ES Lint help with coding convention
  • Feature Detection: Feature detection guarantees that the method is only called if it actually exists and fails gracefully, without any exceptions being thrown, if the method doesn’t exist.
  • Feature Detection can also detect browser support.
  • Alert() can be used for debugging with breakpoints
  • "debugger" command is useful! Learn to use it better.
  • Error Objects: We can create objects to show different types of errors.
  • Types of Error Objects: Error(), EvalError(), RangeError(), ReferenceError(), SyntaxError(), TypeError(), URIError(), InternalError()
  • Throwing Exceptions: Use "throw" to highlight problems in code. It will top the execution.
  • The best practice is to throw an error object. "throw new Error('...')
  • Try, catch, AND finally: Used to catch any errors but not halt the program. It will let us know in the background what the exceptions are.

Red-Green-Refactor:

  1. Write tests (that initially fail)
  2. Write code to pass the tests
  3. Refactor the code
  4. Test refactored code
  5. Write more tests for new features

  • Testing Frameworks: They provide methods that make creating tests easier.
  • Jest is a testing framework developed by Facebook tp do such things.
