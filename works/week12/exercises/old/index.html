Chapter 2: Programming Basics

Programming has evolved from each generation of computers. The computer programs are categorized as low-level programming language that consists of machine code and assembly language. These types of programming languages are somewhat difficult to manipulate thus the invention of High-level programming languages such as Java, Python, JavaScript, or PHP; these programs may use natural human language.

Few details about programming:

  • JavaScript needs only a text editor and a web browser
  • JavaScript was originally called Mocha, but it was changed to LiveScript, then hastily rebranded as JavaScript
  • All programming languages have a list of Reserved Words
  • JavaScript has two types of comments. Single line comments starting with // and Multi-line comments starting with /*and finishing with */
  • The syntax used by JavaScript is known as a C-style syntax because of its similarities with the C programming language.
  • JavaScript has seven different data types. Six of them are primitive data types: String, Symbol, Number, Boolean, Undefined, Null
  • Variables must be declared before being used. The keyword const is used when the variable is not reassigned to another value, whereas let is used if the variable reassignment will take place.
  • A variable can either have a local scope (will only be available inside that block and not be accessible outside of that block) or global scope (accessible everywhere in the program).
  • JS support arithmetic and logical operators
  • Variable values can be changed using increment or decrement operator
  • Type coercion happens when the operands of an operator are of different types.
  • The best way to change a string to a number is to use the Number method, parseInt(), or parseFloat()
  • The preferred way of changing a number to a string is to use the String function or toString()

Chapter 3:Arrays, Logic, and Loops

  • As an ordered list of values, Array syntax can be written as follow:
    • literal: const myArray = []; constructor: const myArray = new Array()
  • Both produce an empty array, but it’s preferable to use array literals (require less typing).
  • An empty array literal is called heroes
  • Operations available with: pop(), push(), shift(), concat(), join(), slice(), splice(), reverse(), sort(), indexof(), includes(),unshift(), add(), delete(), set(), has(), includes()… for example, the delete operator will remove an item from an array
  • A Multidimensional Array is an array of arrays
  • Maps is a data structure as a convenient way of keeping a list of key and value pairs. They are similar to 'hashes', 'hash tables' or 'dictionaries'.

Chapter 4:Functions

  • A function is a chunk of code and in JavaScript, they are considered to be first-class objects
  • Function Expression assigns an anonymous function to a variable
  • A function can also be declared using the constructor Function(). In this way, the body of the function is entered as a string.
  • Parameters and arguments represent values provided for the function as an input. There is a subtle difference though any parameters a function needs are set when the function is defined. When a function is invoked, it is provided with arguments.
  • ES6 introduced a new syntax for declaring functions called the arrow These make declaring functions much more succinct by using less verbose syntax.
  • If the function doesn't require any parameters, a pair of empty parentheses must go before the arrow.
  • Hoisting is the JavaScript interpreter’s action of moving all variable and function declarations to the top of the current scope, regardless of where they are defined. Functions that are defined using a function declaration are automatically hoisted, meaning they can be invoked before they have been defined.
  • A function that is passed as an argument to another is known as a callback.
